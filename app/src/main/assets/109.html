安卓比较常用的距离单位有px，dp，sp。

首先我们介绍一下px，px就是像素的意思，所谓像素就是我们的屏幕分辨率，我手机的分辨率是480x800，就是宽为480个像素，高为800个像素。让我们来新建一个项目，打开xml代码，代码看图，这里我们为了看的更明显一点我们使用 android:background="#ff0000"来更改TextView的背景颜色，android:layout_width="240px"我们设置宽度为240像素，即宽度为手机屏幕的一半android:layout_height="wrap_content"高度为包含内容 android:background="#ff0000"背景颜色为红色，#ff0000代表为红色，具体大家可以百度html颜色代码表对照。android:text="爱上疯人院姑娘"控件的显示内容，注意aide输入中文很蛋疼，我们可以先用拼音代替，最后才改成中文，以免代码错乱，希望aide尽快可以支持中文android:gravity="center"这个是让控件的内容居中，因为默认是左对齐的。

 

　　然后我们按设置键点击run运行我们的项目，看看效果，值得一提的是aide有很强大的预览功能，在我们代码右上角有3个图标，我们点击第一个图片状的图标是预览，这样不需要我们编译即可以看到效果。 大家可以自己多多去试试。

 

　　接下来我们介绍sp，sp通常用来作为字体大小的单位，我们把TextView的宽高都改成包含内容，在代码中加入 android:textSize="30sp" 这个是设置文字大小为30sp 这些基本的属性大家一定要记熟，多写，多练习，以后就不会再向大家做过多解释了，aide的代码补全功能也还是不错的。

 

　　然后我们点击run运行我们的代码，看看效果，用sp的好处是用户在系统设置上更改了字体大小然后返回程序，程序的字体大小也会随之改变，好像要安卓4.0以上才有更改字体大小的设置，我这里是2.3，所以就不向大家做演示了，大家可以自己去试试。

 

　　什么是dpi，dpi为屏幕像素的密度比，换算公式是屏幕宽的平方加屏幕高的平方，然后开根号，即为屏幕对角线的像素，再除以屏幕的尺寸。 什么是 dp？ dp = dip (Device Independent pixels) 换算公式px = dp * (dpi / 160) 在dpi为160的屏幕上: 1dp = 1px 简单来说dp是一种不受像素影响的单位，例如我们在320x480屏幕上，宽度160dp为一半，而在640x960的屏幕上也是一半。 为了迎合多种不同的手机分辨率，我们一般使用sp作为字体大小单位，dp为距离单位。
